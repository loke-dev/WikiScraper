title="REBOL">REBOL</a></td>/n</tr>/n</table>/n<p><b>Forth</b> is an <a href="/wiki/Imperative_programming" title="Imperative programming">imperative</a> <a href="/wiki/Stack-oriented_programming_language" title="Stack-oriented programming language">stack-based</a> computer <a href="/wiki/Programming_language" title="Programming language">programming language</a> and environment originally designed by <a href="/wiki/Charles_H._Moore" title="Charles H. Moore">Charles "Chuck" Moore</a>. Language features include <a href="/wiki/Structured_programming" title="Structured programming">structured programming</a>, <a href="/wiki/Reflection_(computer_science)" class="mw-redirect" title="Reflection (computer science)">reflection</a> (the ability to modify the program structure during program execution), <a href="/wiki/Concatenative_programming_language" title="Concatenative programming language">concatenative programming</a> (functions are composed with juxtaposition) and <a href="/wiki/Extensible_programming" title="Extensible programming">extensibility</a> (the programmer can create new commands). Although not an <a href="/wiki/Acronym" title="Acronym">acronym</a>, the language's name is sometimes spelled with all capital letters as FORTH, following the customary usage during its earlier years.</p>/n<p>A <a href="/wiki/Procedural_programming" title="Procedural programming">procedural programming</a> language without <a href="/wiki/Type_system" title="Type system">type checking</a>, Forth features both interactive execution of commands (making it suitable as a <a href="/wiki/Operating_system_shell" class="mw-redirect" title="Operating system shell">shell</a> for systems that lack a more formal <a href="/wiki/Operating_system" title="Operating system">operating system</a>) and the ability to <a href="/wiki/Compiler" title="Compiler">compile</a> sequences of commands for later execution. Some Forth implementations (usually early versions or those written to be extremely portable) compile <a href="/wiki/Threaded_code" title="Threaded code">threaded code</a>, but many implementations today generate <a href="/wiki/Compiler_optimization" class="mw-redirect" title="Compiler optimization">optimized</a> <a href="/wiki/Machine_code" title="Machine code">machine code</a> like other language compilers.</p>/n<p>Forth is used in the <a href="/wiki/Open_Firmware" title="Open Firmware">Open Firmware</a> <a href="/wiki/Boot_loader" class="mw-redirect" title="Boot loader">boot loader</a>, in <a href="/wiki/Outer_space" title="Outer space">space</a> applications,<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">[1]</a></sup> such as the <a href="/wiki/Philae_(spacecraft)" title="Philae (spacecraft)">Philae spacecraft</a><sup id="cite_ref-2" class="reference"><a href="#cite_note-2">[2]</a></sup><sup id="cite_ref-3" class="reference"><a href="#cite_note-3">[3]</a></sup> and other embedded systems which involve interaction with hardware. The bestselling 1986 <a href="/wiki/DOS" title="DOS">DOS</a> game <i><a href="/wiki/Starflight" title="Starflight">Starflight</a></i>, from <a href="/wiki/Electronic_Arts" title="Electronic Arts">Electronic Arts</a>, was written with a custom Forth.<sup id="cite_ref-4" class="reference"><a href="#cite_note-4">[4]</a></sup></p>/n<p>The free software <a href="/wiki/Gforth" title="Gforth">Gforth</a> implementation is actively maintained, as are several commercially supported systems.</p>/n<p></p>/n<div id="toc" class="toc">/n<div class="toclevel-1 tocsection-27"><a href="#External_links"><span class="tocnumber">12</span> <span class="toctext">External links</span></a></li>/n</ul>/n</div>/n<p></p>/n<h2><span class="mw-headline" id="Overview">Overview</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Forth_(programming_language)&amp;action=edit&amp;section=1" title="Edit section: Overview">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>A Forth environment combines the compiler with an interactive shell, where the user defines and runs <a href="/wiki/Subroutine" title="Subroutine">subroutines</a> called <i>words</i>. Words can be tested, redefined, and debugged as the source is entered without recompiling or restarting the whole program. All syntactic elements, including variables and basic operators, are defined as words. Forth environments vary in how the resulting program is stored, but ideally running the program has the same effect as manually re-entering the source.</p>/n<h3><span class="mw-headline" id="Stacks">Stacks</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Forth_(programming_language)&amp;action=edit&amp;section=2" title="Edit section: Stacks">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<p>Most programming environments with recursive <a href="/wiki/Subroutine" title="Subroutine">subroutines</a> use a <a href="/wiki/Stack_(data_structure)" class="mw-redirect" title="Stack (data structure)">stack</a> for <a href="/wiki/Control_flow" title="Control flow">control flow</a>. This structure typically also stores <a href="/wiki/Local_variable" title="Local variable">local variables</a>, including subroutine <a href="/wiki/Parameter_(computer_science)" class="mw-redirect" title="Parameter (computer science)">parameters</a> (in <a href="/wiki/Call_by_value" class="mw-redirect" title="Call by value">call by value</a> system such as C). Forth often does not have local variables, however, nor is it call-by-value. Instead, intermediate values are kept in another <a href="/wiki/Stack_(data_structure)" class="mw-redirect" title="Stack (data structure)">stack</a>, different from the one it uses for return addresses, loop counters, etc. Words operate directly on the topmost values in the first of these two stacks. It may therefore be called the "parameter" or "data" stack, but most often simply "the" stack. The second, function-call stack is then called the "linkage" or "return" stack, abbreviated <i>rstack</i>. Special rstack manipulation functions provided by the kernel allow it to be used for temporary storage within a word, and it is often used by counted loops, but otherwise it cannot be used to pass parameters or manipulate data.</p>/n<p>Most words are specified in terms of their effect on the stack. Typically, parameters are placed on the top of the stack before the word executes. After execution, the parameters have been erased and replaced with any return values. For arithmetic operators, this follows the rule of <a href="/wiki/Reverse_Polish_notation" title="Reverse Polish notation">reverse Polish notation</a>. See below for examples illustrating stack usage.</p>/n<h3><span class="mw-headline" id="Maintenance">Maintenance</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Forth_(programming_language)&amp;action=edit&amp;section=3" title="Edit section: Maintenance">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<p>Forth is a simple yet extensible language; its modularity and extensibility permit the writing of high-level programs such as <a href="/wiki/Computer-aided_design" title="Computer-aided design">CAD</a> systems. Forth has been used successfully in large, complex projects, while applications developed by competent, disciplined professionals have proven to be easily maintained on evolving hardware platforms over decades of use.<sup id="cite_ref-5" class="reference"><a href="#cite_note-5">[5]</a></sup> Forth has a niche both in astronomical and space applications.<sup id="cite_ref-6" class="reference"><a href="#cite_note-6">[6]</a></sup> Forth is still used today in many <a href="/wiki/Embedded_system" title="Embedded system">embedded systems</a> (small computerized devices) because of its <a href="/wiki/Porting" title="Porting">portability</a>, efficient memory use, short development time, and fast execution speed. It has been implemented efficiently on modern <a href="/wiki/RISC_processor" class="mw-redirect" title="RISC processor">RISC processors</a>, and <a href="/wiki/Stack_machine" title="Stack machine">processors that use Forth as machine language</a> have been produced.<sup id="cite_ref-7" class="reference"><a href="#cite_note-7">[7]</a></sup> Other uses of Forth include the <a href="/wiki/Open_Firmware" title="Open Firmware">Open Firmware</a> <a href="/wiki/Booting" title="Booting">boot ROMs</a> used by <a href="/wiki/Apple_Inc." title="Apple Inc.">Apple</a>, <a href="/wiki/IBM" title="IBM">IBM</a>, <a href="/wiki/Sun_Microsystems" title="Sun Microsystems">Sun</a>, and <a href="/wiki/OLPC_XO-1" class="mw-redirect" title="OLPC XO-1">OLPC XO-1</a>; and the <a rel="nofollow" class="external text" href="http://ficl.sourceforge.net/">FICL</a>-based pre-kernel bootstrap of the <a href="/wiki/FreeBSD" title="FreeBSD">FreeBSD</a> operating system.<sup id="cite_ref-8" class="reference"><a href="#cite_note-8">[8]</a></sup></p>/n<h2><span class="mw-headline" id="History">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Forth_(programming_language)&amp;action=edit&amp;section=4" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>Forth evolved from <a href="/wiki/Charles_H._Moore" title="Charles H. Moore">Charles H. Moore</a>'s personal programming system, which had been in continuous development since 1968.<sup id="cite_ref-evolution_9-0" class="reference"><a href="#cite_note-evolution-9">[9]</a></sup> Forth was first exposed to other programmers in the early 1970s, starting with <a href="/wiki/Elizabeth_Rather" title="Elizabeth Rather">Elizabeth Rather</a> at the US <a href="/wiki/National_Radio_Astronomy_Observatory" title="National Radio Astronomy Observatory">National Radio Astronomy Observatory</a>.<sup id="cite_ref-evolution_9-1" class="reference"><a href="#cite_note-evolution-9">[9]</a></sup> After their work at NRAO, Charles Moore and Elizabeth Rather formed FORTH, Inc. in 1973, refining and porting Forth systems to dozens of other platforms in the next decade.</p>/n<p>Forth is so named because in 1968 "the file holding the interpreter was labeled FOURTH, for 4th (next) generation software—but the <a href="/wiki/IBM_1130" title="IBM 1130">IBM 1130</a> operating system restricted file names to 5 characters."<sup id="cite_ref-10" class="reference"><a href="#cite_note-10">[10]</a></sup> Moore saw Forth as a successor to compile-link-go <a href="/wiki/Third-generation_programming_language" title="Third-generation programming language">third-generation programming languages</a>, or software for "fourth generation" hardware, not a <a href="/wiki/Fourth-generation_programming_language" title="Fourth-generation programming language">fourth-generation programming language</a> as the term has come to be used.</p>/n<p>Because Charles Moore frequently moved from job to job over his career, an early pressure on the developing language was ease of <a href="/wiki/Porting" title="Porting">porting</a> to different computer architectures. A Forth system has often been used to bring up new hardware. For example, Forth was the first resident software on the new <a href="/wiki/Intel_8086" title="Intel 8086">Intel 8086</a> chip in 1978 and MacFORTH was the first resident development system for the first Apple <a href="/wiki/Macintosh" title="Macintosh">Macintosh</a> in 1984.<sup id="cite_ref-evolution_9-2" class="reference"><a href="#cite_note-evolution-9">[9]</a></sup></p>/n<p>FORTH, Inc.'s microFORTH was developed for the <a href="/wiki/Intel_8080" title="Intel 8080">Intel 8080</a>, <a href="/wiki/Motorola_6800" title="Motorola 6800">Motorola 6800</a>, and <a href="/wiki/Zilog_Z80" title="Zilog Z80">Zilog Z80</a> microprocessors starting in 1976. MicroFORTH was later used by hobbyists to generate Forth systems for other architectures, such as the <a href="/wiki/MOS_Technology_6502" title="MOS Technology 6502">6502</a> in 1978. Wide dissemination finally led to standardization of the language. Common practice was codified in the de facto standards FORTH-79<sup id="cite_ref-11" class="reference"><a href="#cite_note-11">[11]</a></sup> and FORTH-83<sup id="cite_ref-12" class="reference"><a href="#cite_note-12">[12]</a></sup> in the years 1979 and 1983, respectively. These standards were unified by <a href="/wiki/American_National_Standards_Institute" title="American National Standards Institute">ANSI</a> in 1994, commonly referred to as ANS Forth.<sup id="cite_ref-13" class="reference"><a href="#cite_note-13">[13]</a></sup><sup id="cite_ref-14" class="reference"><a href="#cite_note-14">[14]</a></sup></p>/n<p>Forth became very popular in the 1980s<sup id="cite_ref-15" class="reference"><a href="#cite_note-15">[15]</a></sup> because it was well suited to the small <a href="/wiki/Microcomputer" title="Microcomputer">microcomputers</a> of that time, being compact and portable. At least one <a href="/wiki/Home_computer" title="Home computer">home computer</a>, the British <a href="/wiki/Jupiter_Ace" title="Jupiter Ace">Jupiter ACE</a>, had Forth in its <a href="/wiki/Read-only_memory" title="Read-only memory">ROM</a>-resident operating system. The <a href="/wiki/Canon_Cat" title="Canon Cat">Canon Cat</a> also used Forth for its system programming, and <a href="/wiki/Rockwell_International" title="Rockwell International">Rockwell</a> produced single-chip microcomputers with resident Forth kernels, the R65F11 and R65F12. A complete family tree is at <a rel="nofollow" class="external text" href="http://www.complang.tuwien.ac.at/forth/family-tree/">TU-Wien</a>. Insoft GraFORTH was a version of Forth with graphics expansions for the Apple II.<sup id="cite_ref-16" class="reference"><a href="#cite_note-16">[16]</a></sup> ASYST was a Forth expansion for measuring and controlling on PCs.<sup id="cite_ref-17" class="reference"><a href="#cite_note-17">[17]</a></sup></p>/n<h2><span id="Programmer's_perspective"></span><span class="mw-headline" id="Programmer.27s_perspective">Programmer's perspective</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Forth_(programming_language)&amp;action=edit&amp;section=5" title="Edit section: Programmer's perspective">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<div role="note" class="hatnote navigation-not-searchable">Further information: <a href="/wiki/Reverse_Polish_notation" title="Reverse Polish notation">Reverse Polish notation</a></div>/n<p>Forth relies heavily on explicit use of a <a href="/wiki/Stack_data_structure" class="mw-redirect" title="Stack data structure">data stack</a> and <a href="/wiki/Reverse_Polish_notation" title="Reverse Polish notation">reverse Polish notation</a> (RPN or postfix notation), commonly used in calculators from <a href="/wiki/Hewlett-Packard" title="Hewlett-Packard">Hewlett-Packard</a>. In RPN, the operator is placed after its operands, as opposed to the more common <a href="/wiki/Infix_notation" title="Infix notation">infix notation</a> where the operator is placed between its operands. Postfix notation makes the language easier to parse and extend; Forth's flexibility makes a static <a href="/wiki/Backus-Naur_form" class="mw-redirect" title="Backus-Naur form">BNF</a> grammar inappropriate, and it does not have a monolithic compiler. Extending the compiler only requires writing a new word, instead of modifying a grammar and changing the underlying implementation.</p>/n<p>Using RPN, one could get the result of the mathematical expression <code>(25 * 10 + 50)</code> this way:</p>/n<div class="mw-highlight mw-content-ltr" dir="ltr">/n<pre>/n <span class="mi">25</span> <span class="mi">10</span> <span class="k">* </span><span class="mi">50</span> <span class="k">+ CR .</span>/n <span class="mi">300</span> <span class="nf">ok</span>/n</pre></div>/n<div class="floatleft"><a href="/wiki/File:Stack1.svg" class="image"><img data-file-width="300" data-file-height="150" /></a></div>/n<p>This command line first puts the numbers 25 and 10 on the implied stack.</p>/n<div style="clear:both;"></div>/n<div class="floatleft"><a multiplies the two numbers on the top of the stack and replaces them with their product.</p>/n<div style="clear:both;"></div>/n<div class="floatleft"><a href="/wiki/File:Forthstack2.svg" stack.</p>/n<div style="clear:both;"></div>/n<div class="floatleft"><a href="/wiki/File:Forthstack3.svg" class="image"><img alt="Forthstack3.svg" to the previous product. The CR moves the output to a new line (it is only for formatting purposes and could be omitted but—in most implementations—without it the output would occur on the same line as the input and would be less readable in the example). Finally, the <code>.</code> command prints the result to the user's terminal. As everything has completed successfully at that point, the text interpreter then outputs the prompt "ok" and moves to a new line to get more input without needing anything explicit to do that.<sup id="cite_ref-18" class="reference"><a href="#cite_note-18">[18]</a></sup></p>/n<div style="clear:both;"></div>/n<p>Even Forth's structural features are stack-based. For example:</p>/n<div class="mw-highlight mw-content-ltr" dir="ltr">/n<pre>/n <span class="kn">:</span> <span class="nc">FLOOR5</span> <span class="c1">( n -- n' )</span>   <span class="k">DUP </span><span class="mi">6</span> <span class="k">&lt; IF DROP </span><span class="mi">5</span> <span class="k">ELSE </span><span class="mi">1</span> <span class="k">- THEN ;</span>/n</pre></div>/n<p>This code defines a new word (again, <i>word</i> is the term used for a subroutine) called <code>FLOOR5</code> using the following commands: <code>DUP</code> duplicates the number on the stack; <code>6</code> places a 6 on top of the stack; <code>&lt;</code> compares the top two numbers on the stack (6 and the <code>DUP</code>ed input), and replaces them with a true-or-false value; <code>IF</code> takes a true-or-false value and chooses to execute commands immediately after it or to skip to the <code>ELSE</code>; <code>DROP</code> discards the value on the stack; and <code>THEN</code> ends the conditional. The text in parentheses is a comment, advising that this word expects a number on the stack and will return a possibly changed number. The <code>FLOOR5</code> word is equivalent to this function written in the <a href="/wiki/C_(programming_language)" title="C (programming language)">C programming language</a> using the <a href="/wiki/%3F:" title="?:">ternary operator</a>:</p>/n<div class="mw-highlight mw-content-ltr" dir="ltr">/n<pre>/n<span class="kt">int</span> <span class="nf">floor5</span><span class="p">(</span><span class="kt">int</span> <span class="n">v</span><span class="p">)</span> <span class="p">{</span>/n  <span class="k">return</span> <span class="p">(</span><span class="n">v</span> <span class="o">&lt;</span> <span class="mi">6</span><span class="p">)</span> <span class="o">?</span> <span class="mi">5</span> <span class="o">:</span> <span class="p">(</span><span class="n">v</span> <span class="o">-</span> <span class="mi">1</span><span class="p">);</span>/n<span class="p">}</span>/n</pre></div>/n<p>This function is written more succinctly as:</p>/n<div class="mw-highlight mw-content-ltr" dir="ltr">/n<pre>/n <span class="kn">:</span> <span class="nc">FLOOR5</span> <span class="c1">( n -- n' )</span> <span class="k">1- </span><span class="mi">5</span> <span class="k">MAX ;</span>/n</pre></div>/n<p>You could run this word as follows:</p>/n<div class="mw-highlight mw-content-ltr" dir="ltr">/n<pre>/n <span class="mi">1</span> <span class="nf">FLOOR5</span> <span class="k">CR . </span>/n <span class="mi">5</span> <span class="nf">ok</span>/n <span class="mi">8</span> <span class="nf">FLOOR5</span> <span class="k">CR .</span>/n <span class="mi">7</span> <span class="nf">ok</span>/n</pre></div>/n<p>First the interpreter pushes a number (1 or 8) onto the stack, then it calls FLOOR5, which pops off this number again and pushes the result. The CR moves the output to a new line (again, this is only here for readability). Finally, a call to "." pops the result and prints it to the user's terminal.</p>/n<h2><span class="mw-headline" id="Facilities">Facilities</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Forth_(programming_language)&amp;action=edit&amp;section=6" title="Edit section: Facilities">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>Forth has no explicit <a href="/wiki/Formal_grammar" title="Formal grammar">grammar</a>. The interpreter reads a line of input from the user input device, which is then parsed for a word using spaces as a <a href="/wiki/Delimiter" title="Delimiter">delimiter</a>; some systems recognise additional <a href="/wiki/Whitespace_(computer_science)" class="mw-redirect" title="Whitespace (computer science)">whitespace</a> characters. When the interpreter finds a word, it looks the word up in the <i>dictionary</i>. If the word is found, the interpreter executes the code associated with the word, and then returns to parse the rest of the input stream. If the word isn't found, the word is assumed to be a number and an attempt is made to convert it into a number and push it on the stack; if successful, the interpreter continues parsing the input stream. Otherwise, if both the lookup and the number conversion fail, the interpreter prints the word followed by an error message indicating the word is not recognised, flushes the input stream, and waits for new user input.<sup id="cite_ref-19" class="reference"><a href="#cite_note-19">[19]</a></sup></p>/n<p>The definition of a new word is started with the word <code>:</code> (colon) and ends with the word <code>;</code> (semi-colon). For example,</p>/n<div class="mw-highlight mw-content-ltr" dir="ltr">/n<pre>/n <span class="kn">:</span> <span class="nc">X</span> <span class="k">DUP 1+ . . ;</span>/n</pre></div>/n<p>will compile the word <code>X</code>, and makes the name findable in the dictionary. When executed by typing <code>10 X</code> at the console this will print <code>11 10</code>.<sup id="cite_ref-compiler_20-0" class="reference"><a href="#cite_note-compiler-20">[20]</a></sup></p>/n<p>Most Forth systems include an <a href="/wiki/Assembly_language#Assembler" title="Assembly language">assembler</a> that allows one to specify words using the processor's facilities at its lowest level. Mostly the assembler is tucked away in a separate <a href="/wiki/Namespace" title="Namespace">namespace</a> (<i>wordlist</i>) as relatively few users want to use it. Forth assemblers may use a reverse-polish syntax in which the parameters of an instruction precede the instruction, but designs vary widely and are specific to the Forth implementation. A typical reverse-polish assembler prepares the operands on the stack and have the mnemonic copy the whole instruction into memory as the last step. A Forth assembler is by nature a macro assembler, so that it is easy to define an alias for registers according to their role in the Forth system: e.g. "datastackpointer" for the register used as a stack pointer.<sup id="cite_ref-21" class="reference"><a href="#cite_note-21">[21]</a></sup></p>/n<h3><span id="Operating_system,_files,_and_multitasking"></span><span class="mw-headline" id="Operating_system.2C_files.2C_and_multitasking">Operating system, files, and multitasking</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Forth_(programming_language)&amp;action=edit&amp;section=7" title="Edit section: Operating system, files, and multitasking">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<p>Most Forth systems run under a host operating system such as <a href="/wiki/Microsoft_Windows" title="Microsoft Windows">Microsoft Windows</a>, <a href="/wiki/Linux" title="Linux">Linux</a> or a version of <a href="/wiki/Unix" title="Unix">Unix</a> and use the host operating system's file system for source and data files; the ANSI Forth Standard describes the words used for I/O. All modern Forth systems use normal text files for source, even if they are embedded. An embedded system with a resident compiler gets its source via a serial line.</p>/n<p>Classic Forth systems traditionally use neither <a href="/wiki/Operating_system" title="Operating system">operating system</a> nor <a href="/wiki/File_system" title="File system">file system</a>. Instead of storing code in files, source code is stored in disk blocks written to physical disk addresses. The word <code>BLOCK</code> is employed to translate the number of a 1K-sized block of disk space into the address of a buffer containing the data, which is managed automatically by the Forth system. Block use has become rare since the mid-1990s. In a hosted system those blocks too are allocated in a normal file in any case.</p>/n<p><a href="/wiki/Computer_multitasking" title="Computer multitasking">Multitasking</a>, most commonly <a href="/wiki/Computer_multitasking#Cooperative_multitasking.2Ftime-sharing" title="Computer multitasking">cooperative</a> <a href="/wiki/Round-robin_scheduling" title="Round-robin scheduling">round-robin scheduling</a>, is normally available (although multitasking words and support are not covered by the ANSI Forth Standard). The word <code>PAUSE</code> is used to save the current task's execution context, to locate the next task, and restore its execution context. Each task has its own stacks, private copies of some control variables and a scratch area. Swapping tasks is simple and efficient; as a result, Forth multitaskers are available even on very simple <a href="/wiki/Microcontroller" title="Microcontroller">microcontrollers</a>, such as the <a href="/wiki/Intel_MCS-51" title="Intel MCS-51">Intel 8051</a>, <a href="/wiki/Atmel_AVR" title="Atmel AVR">Atmel AVR</a>, and <a href="/wiki/TI_MSP430" title="TI MSP430">TI MSP430</a>.<sup id="cite_ref-22" class="reference"><a href="#cite_note-22">[22]</a></sup></p>/n<p>Other non-standard facilities include a mechanism for issuing <a href="/wiki/System_call" title="System call">calls</a> to the host OS or <a href="/wiki/Windowing_system" title="Windowing system">windowing systems</a>, and many provide extensions that employ the scheduling provided by the operating system. Typically they have a larger and different set of words from the stand-alone Forth's <code>PAUSE</code> word for task creation, suspension, destruction and modification of priority.</p>/n<h3><span class="mw-headline" id="Self-compilation_and_cross_compilation">Self-compilation and cross compilation</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Forth_(programming_language)&amp;action=edit&amp;section=8" title="Edit section: Self-compilation and cross compilation">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<p>A fully featured Forth system with all source code will compile itself, a technique commonly called meta-compilation by Forth programmers (although the term doesn't exactly match <a href="/wiki/Meta-Compilation" class="mw-redirect" title="Meta-Compilation">meta-compilation</a> as it is normally defined). The usual method is to redefine the handful of words that place compiled bits into memory. The compiler's words use specially named versions of fetch and store that can be redirected to a buffer area in memory. The buffer area simulates or accesses a memory area beginning at a different address than the code buffer. Such compilers define words to access both the target computer's memory, and the host (compiling) computer's memory.<sup id="cite_ref-23" class="reference"><a href="#cite_note-23">[23]</a></sup></p>/n<p>After the fetch and store operations are redefined for the code space, the compiler, assembler, etc. are recompiled using the new definitions of fetch and store. This effectively reuses all the code of the compiler and interpreter. Then, the Forth system's code is compiled, but this version is stored in the buffer. The buffer in memory is written to disk, and ways are provided to load it temporarily into memory for testing. When the new version appears to work, it is written over the previous version.</p>/n<p>Numerous variations of such compilers exist for different environments. For <a href="/wiki/Embedded_system" title="Embedded system">embedded systems</a>, the code may instead be written to another computer, a technique known as <a href="/wiki/Cross-compilation" class="mw-redirect" title="Cross-compilation">cross compilation</a>, over a serial port or even a single <a href="/wiki/Transistor-transistor_logic" class="mw-redirect" title="Transistor-transistor logic">TTL</a> bit, while keeping the word names and other non-executing parts of the dictionary in the original compiling computer. The minimum definitions for such a Forth compiler are the words that fetch and store a byte, and the word that commands a Forth word to be executed. Often the most time-consuming part of writing a remote port is constructing the initial program to implement fetch, store and execute, but many modern microprocessors have integrated debugging features (such as the <a href="/wiki/Motorola_CPU32" class="mw-redirect" title="Motorola CPU32">Motorola CPU32</a>) that eliminate this task.<sup id="cite_ref-24" class="reference"><a href="#cite_note-24">[24]</a></sup></p>/n<h2><span class="mw-headline" id="Structure_of_the_language">Structure of the language</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Forth_(programming_language)&amp;action=edit&amp;section=9" title="Edit section: Structure of the language">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>The basic data structure of Forth is the "dictionary" which maps "words" to executable code or named data structures. The dictionary is laid out in memory as a tree of <a href="/wiki/Linked_list" title="Linked list">linked lists</a> with the links proceeding from the latest (most recently) defined word to the oldest, until a <a href="/wiki/Sentinel_value" title="Sentinel value">sentinel value</a>, usually a NULL pointer, is found. A context switch causes a list search to start at a different leaf. A linked list search continues as the branch merges into the main trunk leading eventually back to the sentinel, the root. There can be several dictionaries. In rare cases such as meta-compilation a dictionary might be isolated and stand-alone. The effect resembles that of nesting namespaces and can overload keywords depending on the context.</p>/n<p>A defined word generally consists of <i>head</i> and <i>body</i> with the head consisting of the <i>name field</i> (NF) and the <i>link field</i> (LF) and body consisting of the <i>code field</i> (CF) and the <i>parameter field</i> (PF).</p>/n<p>Head and body of a dictionary entry are treated separately because they may not be contiguous. For example, when a Forth program is recompiled for a new platform, the head may remain on the compiling computer, while the body goes to the new platform. In some environments (such as <a href="/wiki/Embedded_system" title="Embedded system">embedded systems</a>) the heads occupy memory unnecessarily. However, some cross-compilers may put heads in the target if the target itself is expected to support an interactive Forth.<sup id="cite_ref-25" class="reference"><a href="#cite_note-25">[25]</a></sup></p>/n<h3><span class="mw-headline" id="Dictionary_entry">Dictionary entry</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Forth_(programming_language)&amp;action=edit&amp;section=10" title="Edit section: Dictionary entry">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<p>The exact format of a dictionary entry is not prescribed, and implementations vary. However, certain components are almost always present, though the exact size and order may vary. Described as a structure, a dictionary entry might look this way:<sup id="cite_ref-26" class="reference"><a href="#cite_note-26">[26]</a></sup></p>/n<div class="mw-highlight mw-content-ltr" dir="ltr">/n<pre>/n <span class="nf">structure</span>/n   <span class="nf">byte:</span>       <span class="nf">flag</span>           <span class="c1">\ 3bit flags + length of word's name</span>/n   <span class="nf">char-array:</span> <span class="nf">name</span>           <span class="c1">\ name's runtime length isn't known at compile time</span>/n   <span class="nf">address:</span>    <span class="k">previous </span>      <span class="c1">\ link field, backward ptr to previous word</span>/n   <span class="nf">address:</span>    <span class="nf">codeword</span>       <span class="c1">\ ptr to the code to execute this word</span>/n   <span class="nf">any-array:</span>  <span class="nf">parameterfield</span> <span class="c1">\ unknown length of data, words, or opcodes</span>/n <span class="k">end-structure </span><span class="nf">forthword</span>/n</pre></div>/n<p>The name field starts with a prefix giving the length of the word's name (typically up to 32 bytes), and several bits for flags. The character representation of the word's name then follows the prefix. Depending on the particular implementation of Forth, there may be one or more NUL ('\0') bytes for alignment.</p>/n<p>The link field contains a pointer to the previously defined word. The pointer may be a relative displacement or an absolute address that points to the next oldest sibling.</p>/n<p>The code field pointer will be either the address of the word which will execute the code or data in the parameter field or the beginning of machine code that the processor will execute directly. For colon defined words, the code field pointer points to the word that will save the current Forth instruction pointer (IP) on the return stack, and load the IP with the new address from which to continue execution of words. This is the same as what a processor's call/return instructions do.</p>/n<h3><span class="mw-headline" id="Structure_of_the_compiler">Structure of the compiler</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Forth_(programming_language)&amp;action=edit&amp;section=11" title="Edit section: Structure of the compiler">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<p>The compiler itself is not a monolithic program. It consists of Forth words visible to the system, and usable by a programmer. This allows a programmer to change the compiler's words for special purposes.</p>/n<p>The "compile time" flag in the name field is set for words with "compile time" behavior. Most simple words execute the same code whether they are typed on a command line, or embedded in code. When compiling these, the compiler simply places code or a threaded pointer to the word.<sup id="cite_ref-compiler_20-1" class="reference"><a href="#cite_note-compiler-20">[20]</a></sup></p>/n<p>The classic examples of compile-time words are the <a href="/wiki/Control_structure" class="mw-redirect" title="Control structure">control structures</a> such as <code>IF</code> and <code>WHILE</code>. Almost all of Forth's control structures and almost all of its compiler are implemented as compile-time words. Apart from some rarely used <a href="/wiki/Control_flow" title="Control flow">control flow</a> words only found in a few implementations, such as a conditional return, all of Forth's <a href="/wiki/Control_flow" title="Control flow">control flow</a> words are executed during compilation to compile various combinations of primitive words along with their branch addresses. For instance, <code>IF</code> and <code>WHILE</code>, and the words that match with those, set up <code>BRANCH</code> (unconditional branch) and <code>?BRANCH</code> (pop a value off the stack, and branch if it is false). Counted loop <a href="/wiki/Control_flow" title="Control flow">control flow</a> words work similarly but set up combinations of primitive words that work with a counter, and so on. During compilation, the data stack is used to support control structure balancing, nesting, and back-patching of branch addresses. The snippet:</p>/n<div class="mw-highlight mw-content-ltr" dir="ltr">/n<pre>/n <span class="kt">...</span> <span class="k">DUP </span><span class="mi">6</span> <span class="k">&lt; IF DROP </span><span class="mi">5</span> <span class="k">ELSE </span><span class="mi">1</span> <span class="k">- THEN </span><span class="kt">...</span>/n</pre></div>/n<p>would be compiled to the following sequence inside a definition:</p>/n<div class="mw-highlight mw-content-ltr" dir="ltr">/n<pre>/n <span class="kt">...</span> <span class="k">DUP </span><span class="nf">LIT</span> <span class="mi">6</span> <span class="k">&lt; </span><span class="nf">?BRANCH</span> <span class="mi">5</span>  <span class="k">DROP </span><span class="nf">LIT</span> <span class="mi">5</span>  <span class="nf">BRANCH</span> <span class="mi">3</span>  <span class="nf">LIT</span> <span class="mi">1</span> <span class="k">- </span><span class="kt">...</span>/n</pre></div>/n<p>The numbers after <code>BRANCH</code> represent relative jump addresses. <code>LIT</code> is the primitive word for pushing a "literal" number onto the data stack.</p>/n<h4><span class="mw-headline" id="Compilation_state_and_interpretation_state">Compilation state and interpretation state</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Forth_(programming_language)&amp;action=edit&amp;section=12" title="Edit section: Compilation state and interpretation state">edit</a><span class="mw-editsection-bracket">]</span></span></h4>/n<p>The word <code>:</code> (colon) parses a name as a parameter, creates a dictionary entry (a <i>colon definition</i>) and enters compilation state. The interpreter continues to read space-delimited words from the user input device. If a word is found, the interpreter executes the <i>compilation semantics</i> associated with the word, instead of the <i>interpretation semantics</i>. The default compilation semantics of a word are to append its interpretation semantics to the current definition.<sup id="cite_ref-compiler_20-2" class="reference"><a href="#cite_note-compiler-20">[20]</a></sup></p>/n<p>The word <code>;</code> (semi-colon) finishes the current definition and returns to interpretation state. It is an example of a word whose compilation semantics differ from the default. The interpretation semantics of <code>;</code> (semi-colon), most control flow words, and several other words are undefined in ANS Forth, meaning that they must only be used inside of definitions and not on the interactive command line.<sup id="cite_ref-compiler_20-3" class="reference"><a href="#cite_note-compiler-20">[20]</a></sup></p>/n<p>The interpreter state can be changed manually with the words <code>[</code> (left-bracket) and <code>]</code> (right-bracket) which enter interpretation state or compilation state, respectively. These words can be used with the word <code>LITERAL</code> to calculate a value during a compilation and to insert the calculated value into the current colon definition. <code>LITERAL</code> has the compilation semantics to take an object from the data stack and to append semantics to the current colon definition to place that object on the data stack.</p>/n<p>In ANS Forth, the current state of the interpreter can be read from the <a href="/wiki/Flag_(computing)" class="mw-redirect" title="Flag (computing)">flag</a> <code>STATE</code> which contains the value true when in compilation state and false otherwise. This allows the implementation of so-called <i>state-smart words</i> with behavior that changes according to the current state of the interpreter.</p>/n<h4><span class="mw-headline" id="Immediate_words">Immediate words</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Forth_(programming_language)&amp;action=edit&amp;section=13" title="Edit section: Immediate words">edit</a><span class="mw-editsection-bracket">]</span></span></h4>/n<p>The word <code>IMMEDIATE</code> marks the most recent colon definition as an <i>immediate word</i>, effectively replacing its compilation semantics with its interpretation semantics.<sup id="cite_ref-27" class="reference"><a href="#cite_note-27">[27]</a></sup> Immediate words are normally executed during compilation, not compiled but this can be overridden by the programmer, in either state. <code>;</code> is an example of an immediate word. In ANS Forth, the word <code>POSTPONE</code> takes a name as a parameter and appends the compilation semantics of the named word to the current definition even if the word was marked immediate. Forth-83 defined separate words <code>COMPILE</code> and <code>[COMPILE]</code> to force the compilation of non-immediate and immediate words, respectively.</p>/n<h4><span class="mw-headline" id="Unnamed_words_and_execution_tokens">Unnamed words and execution tokens</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Forth_(programming_language)&amp;action=edit&amp;section=14" title="Edit section: Unnamed words and execution tokens">edit</a><span class="mw-editsection-bracket">]</span></span></h4>/n<p>In ANS Forth, unnamed words can be defined with the word <code>:NONAME</code> which compiles the following words up to the next <code>;</code> (semi-colon) and leaves an <i>execution token</i> on the data stack. The execution token provides an opaque handle for the compiled semantics, similar to the <a href="/wiki/Function_pointer" title="Function pointer">function pointers</a> of the <a href="/wiki/C_(programming_language)" title="C (programming language)">C programming language</a>.</p>/n<p>Execution tokens can be stored in variables. The word <code>EXECUTE</code> takes an execution token from the data stack and performs the associated semantics. The word <code>COMPILE,</code> (compile-comma) takes an execution token from the data stack and appends the associated semantics to the current definition.</p>/n<p>The word <code>'</code> (tick) takes the name of a word as a parameter and returns the execution token associated with that word on the data stack. In interpretation state, <code>' RANDOM-WORD EXECUTE</code> is equivalent to <code>RANDOM-WORD</code>.<sup id="cite_ref-28" class="reference"><a href="#cite_note-28">[28]</a></sup></p>/n<h4><span class="mw-headline" id="Parsing_words_and_comments">Parsing words and comments</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Forth_(programming_language)&amp;action=edit&amp;section=15" title="Edit section: Parsing words and comments">edit</a><span class="mw-editsection-bracket">]</span></span></h4>/n<p>The words <code>:</code> (colon), <code>POSTPONE</code>, <code>'</code> (tick) are examples of <i>parsing words</i> that take their arguments from the user input device instead of the data stack. Another example is the word <code>(</code> (paren) which reads and ignores the following words up to and including the next right parenthesis and is used to place comments in a colon definition. Similarly, the word <code>\</code> (backslash) is used for comments that continue to the end of the current line. To be parsed correctly, <code>(</code> (paren) and <code>\</code> (backslash) must be separated by whitespace from the following comment text.</p>/n<h3><span class="mw-headline" id="Structure_of_code">Structure of code</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Forth_(programming_language)&amp;action=edit&amp;section=16" title="Edit section: Structure of code">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<p>In most Forth systems, the body of a code definition consists of either <a href="/wiki/Machine_language" class="mw-redirect" title="Machine language">machine language</a>, or some form of <a href="/wiki/Threaded_code" title="Threaded code">threaded code</a>. The original Forth which follows the informal FIG standard (Forth Interest Group), is a TIL (Threaded Interpretive Language). This is also called indirect-threaded code, but direct-threaded and subroutine threaded Forths have also become popular in modern times. The fastest modern Forths use subroutine threading, insert simple words as macros, and perform <a href="/wiki/Peephole_optimization" title="Peephole optimization">peephole optimization</a> or other optimizing strategies to make the code smaller and faster.<sup id="cite_ref-29" class="reference"><a href="#cite_note-29">[29]</a></sup></p>/n<h3><span class="mw-headline" id="Data_objects">Data objects</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Forth_(programming_language)&amp;action=edit&amp;section=17" title="Edit section: Data objects">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<p>When a word is a variable or other data object, the CF points to the runtime code associated with the defining word that created it. A defining word has a characteristic "defining behavior" (creating a dictionary entry plus possibly allocating and initializing data space) and also specifies the behavior of an instance of the class of words constructed by this defining word. Examples include:</p>/n<dl>/n<dt><code>VARIABLE</code></dt>/n<dd>Names an uninitialized, one-cell memory location. Instance behavior of a <code>VARIABLE</code> returns its address on the stack.</dd>/n<dt><code>CONSTANT</code></dt>/n<dd>Names a value (specified as an argument to <code>CONSTANT</code>). Instance behavior returns the value.</dd>/n<dt><code>CREATE</code></dt>/n<dd>Names a location; space may be allocated at this location, or it can be set to contain a string or other initialized value. Instance behavior returns the address of the beginning of this space.</dd>/n</dl>/n<p>Forth also provides a facility by which a programmer can define new application-specific defining words, specifying both a custom defining behavior and instance behavior. Some examples include circular buffers, named bits on an I/O port, and automatically indexed arrays.</p>/n<p>Data objects defined by these and similar words are global in scope. The function provided by local variables in other languages is provided by the data stack in Forth (although Forth also has real local variables). Forth programming style uses very few named data objects compared with other languages; typically such data objects are used to contain data which is used by a number of words or tasks (in a multitasked implementation).<sup id="cite_ref-30" class="reference"><a href="#cite_note-30">[30]</a></sup></p>/n<p>Forth does not enforce consistency of data type usage; it is the programmer's responsibility to use appropriate operators to fetch and store values or perform other operations on data.</p>/n<h2><span class="mw-headline" id="Programming">Programming</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Forth_(programming_language)&amp;action=edit&amp;section=18" title="Edit section: Programming">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>Words written in Forth are compiled into an executable form. The classical "indirect threaded" implementations compile lists of addresses of words to be executed in turn; many modern systems generate actual machine code (including calls to some external words and code for others expanded in place). Some systems have optimizing compilers. Generally speaking, a Forth program is saved as the memory image of the compiled program with a single command (e.g., RUN) that is executed when the compiled version is loaded.</p>/n<p>During development, the programmer uses the interpreter in <a href="/wiki/REPL" class="mw-redirect" title="REPL">REPL</a> mode to execute and test each little piece as it is developed. Most Forth programmers therefore advocate a loose top-down design, and bottom-up development with continuous testing and integration.<sup id="cite_ref-31" class="reference"><a href="#cite_note-31">[31]</a></sup></p>/n<p>The top-down design is usually separation of the program into "vocabularies" that are then used as high-level sets of tools to write the final program. A well-designed Forth program reads like natural language, and implements not just a single solution, but also sets of tools to attack related problems.<sup id="cite_ref-32" class="reference"><a href="#cite_note-32">[32]</a></sup></p>/n<h2><span class="mw-headline" id="Code_examples">Code examples</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Forth_(programming_language)&amp;action=edit&amp;section=19" title="Edit section: Code examples">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<h3><span class="mw-headline" id="Hello_world">Hello world</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Forth_(programming_language)&amp;action=edit&amp;section=20" title="Edit section: Hello world">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<div role="note" class="hatnote navigation-not-searchable">For an explanation of the tradition of programming "Hello, World!", see <a href="/wiki/%22Hello,_World!%22_program" title="&quot;Hello, World!&quot; program">"Hello, World!" program</a>.</div>/n<p>One possible implementation:</p>/n<div class="mw-highlight mw-content-ltr" dir="ltr">/n<pre>/n <span class="kn">:</span> <span class="nc">HELLO</span>  <span class="c1">( -- )</span>  <span class="k">CR </span><span class="s">."</span> <span class="s">Hello, world!</span><span class="nf">"</span> <span class="k">;</span>/n</pre></div>/n<pre>/nHELLO &lt;cr&gt;/nHello, world!/n</pre>/n<p>The word <code>CR</code> (Carriage Return) causes the following output to be displayed on a new line. The parsing word <code>."</code> (dot-quote) reads a double-quote delimited string and appends code to the current definition so that the parsed string will be displayed on execution. The space character separating the word <code>."</code> from the string <code>Hello, world!</code> is not included as part of the string. It is needed so that the parser recognizes <code>."</code> as a Forth word.</p>/n<p>A standard Forth system is also an interpreter, and the same output can be obtained by typing the following code fragment into the Forth console:</p>/n<div class="mw-highlight mw-content-ltr" dir="ltr">/n<pre>/n <span class="k">CR </span><span class="kt">.</span><span class="c1">( Hello, world!)</span>/n</pre></div>/n<p><code>.(</code> (dot-paren) is an immediate word that parses a parenthesis-delimited string and displays it. As with the word <code>."</code> the space character separating <code>.(</code> from <code>Hello, world!</code> is not part of the string.</p>/n<p>The word <code>CR</code> comes before the text to print. By convention, the Forth interpreter does not start output on a new line. Also by convention, the interpreter waits for input at the end of the previous line, after an <code>ok</code> prompt. There is no implied "flush-buffer" action in Forth's <code>CR</code>, as sometimes is in other programming languages.</p>/n<h3><span class="mw-headline" id="Mixing_states_of_compiling_and_interpreting">Mixing states of compiling and interpreting</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Forth_(programming_language)&amp;action=edit&amp;section=21" title="Edit section: Mixing states of compiling and interpreting">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<p>Here is the definition of a word <code>EMIT-Q</code> which when executed emits the single character <code>Q</code>:</p>/n<div class="mw-highlight mw-content-ltr" dir="ltr">/n<pre>/n <span class="kn">:</span> <span class="nc">EMIT-Q</span>   <span class="mi">81</span> <span class="c1">( the ASCII value for the character 'Q' )</span> <span class="k">EMIT ;</span>/n</pre></div>/n<p>This definition was written to use the <a href="/wiki/ASCII" title="ASCII">ASCII</a> value of the <code>Q</code> character (81) directly. The text between the parentheses is a comment and is ignored by the compiler. The word <code>EMIT</code> takes a value from the data stack and displays the corresponding character.</p>/n<p>The following redefinition of <code>EMIT-Q</code> uses the words <code>[</code> (left-bracket), <code>]</code> (right-bracket), <code>CHAR</code> and <code>LITERAL</code> to temporarily switch to interpreter state, calculate the ASCII value of the <code>Q</code> character, return to compilation state and append the calculated value to the current colon definition:</p>/n<div class="mw-highlight mw-content-ltr" dir="ltr">/n<pre>/n <span class="kn">:</span> <span class="nc">EMIT-Q</span>   <span class="nf">[</span> <span class="k">CHAR </span><span class="nf">Q</span> <span class="nf">]</span>  <span class="k">LITERAL </span> <span class="k">EMIT ;</span>/n</pre></div>/n<p>The parsing word <code>CHAR</code> takes a space-delimited word as parameter and places the value of its first character on the data stack.
