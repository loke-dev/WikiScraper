language)">Scala</a></td>/n</tr>/n</table>/n<p><b>ML</b> ('Meta Language') is a general-purpose <a href="/wiki/Functional_programming_language" class="mw-redirect" title="Functional programming language">functional programming language</a>. It has roots in <a href="/wiki/Lisp_(programming_language)" title="Lisp (programming language)">Lisp</a>, and has been characterized as "Lisp with types". It is known for its use of the polymorphic <a href="/wiki/Hindley%E2%80%93Milner_type_system" title="Hindley–Milner type system">Hindley–Milner type system</a>, which automatically assigns the <a href="/wiki/Data_type" title="Data type">types</a> of most <a href="/wiki/Expression_(programming)" class="mw-redirect" title="Expression (programming)">expressions</a> without requiring explicit type annotations, and ensures type safety&#160;– there is a formal proof that a well-typed ML program does not cause runtime type errors.<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">[1]</a></sup> ML provides pattern matching for function arguments, <a href="/wiki/Garbage_collection_(computer_science)" title="Garbage collection (computer science)">garbage collection</a>, <a href="/wiki/Imperative_programming" title="Imperative programming">imperative programming</a>, <a href="/wiki/Call-by-value" class="mw-redirect" title="Call-by-value">call-by-value</a> and <a href="/wiki/Currying" title="Currying">currying</a>. It is used heavily in programming language research and is one of the few languages to be completely specified and verified using <a href="/wiki/Formal_semantics_of_programming_languages" class="mw-redirect" title="Formal semantics of programming languages">formal semantics</a>. Its types and pattern matching make it well-suited and commonly used to operate on other formal languages, such as in <a href="/wiki/Compiler_writing" class="mw-redirect" title="Compiler writing">compiler writing</a>, <a href="/wiki/Automated_theorem_proving" title="Automated theorem proving">automated theorem proving</a> and <a href="/wiki/Formal_verification" title="Formal verification">formal verification</a>.</p>/n<p></p>/n<div id="toc" class="toc">/n<div class="toctitle">/n<h2>Contents</h2>/n</div>/n<ul>/n<li class="toclevel-1 tocsection-1"><a href="#Overview"><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=ML_(programming_language)&amp;action=edit&amp;section=1" title="Edit section: Overview">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>Features of ML include a call-by-value <a href="/wiki/Evaluation_strategy" title="Evaluation strategy">evaluation strategy</a>, <a href="/wiki/First-class_function" title="First-class function">first-class functions</a>, automatic memory management through <a href="/wiki/Garbage_collection_(computer_science)" title="Garbage collection (computer science)">garbage collection</a>, <a href="/wiki/Parametric_polymorphism" title="Parametric polymorphism">parametric polymorphism</a>, <a href="/wiki/Type_system#Static_typing" title="Type system">static typing</a>, <a href="/wiki/Type_inference" title="Type inference">type inference</a>, <a href="/wiki/Algebraic_data_types" class="mw-redirect" title="Algebraic data types">algebraic data types</a>, <a href="/wiki/Pattern_matching" title="Pattern matching">pattern matching</a>, and <a href="/wiki/Exception_handling" title="Exception handling">exception handling</a>. ML uses <a href="/wiki/Scope_(computer_science)#Lexical_scoping" title="Scope (computer science)">static scoping</a> rules.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (May 2015)">citation needed</span></a></i>]</sup></p>/n<p>ML can be referred to as an <i>impure</i> functional language, because although it encourages functional programming, it does allow <a href="/wiki/Side-effect_(computer_science)" class="mw-redirect" title="Side-effect (computer science)">side-effects</a> (like languages such as <a href="/wiki/Lisp_(programming_language)" title="Lisp (programming language)">Lisp</a>, but unlike a <a href="/wiki/Purely_functional_language" class="mw-redirect" title="Purely functional language">purely functional language</a> such as <a href="/wiki/Haskell_(programming_language)" title="Haskell (programming language)">Haskell</a>). Like most programming languages, ML uses <a href="/wiki/Eager_evaluation" title="Eager evaluation">eager evaluation</a>, meaning that all subexpressions are always evaluated, though <a href="/wiki/Lazy_evaluation" title="Lazy evaluation">lazy evaluation</a> can be achieved through the use of <a href="/wiki/Closure_(computer_science)" class="mw-redirect" title="Closure (computer science)">closures</a>. Thus one can create and use infinite streams as in Haskell, but their expression is indirect.</p>/n<p>ML's strengths are mostly applied in language design and manipulation (compilers, analyzers, theorem provers), but it is a general-purpose language also used in bioinformatics, financial systems, and applications including a genealogical database and a <a href="/wiki/MLDonkey" title="MLDonkey">peer-to-peer client/server program</a>.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (April 2013)">citation needed</span></a></i>]</sup></p>/n<p>ML was developed by <a href="/wiki/Robin_Milner" title="Robin Milner">Robin Milner</a> and others in the early 1970s at the <a href="/wiki/University_of_Edinburgh" title="University of Edinburgh">University of Edinburgh</a>,<sup id="cite_ref-Gordon1996_2-0" class="reference"><a href="#cite_note-Gordon1996-2">[2]</a></sup> whose syntax is inspired by <a href="/wiki/ISWIM" title="ISWIM">ISWIM</a>. Historically, ML was conceived to develop proof tactics in the <a href="/wiki/Logic_for_Computable_Functions" title="Logic for Computable Functions">LCF theorem prover</a> (whose language, <i>pplambda</i>, a combination of the <a href="/wiki/First-order_logic" title="First-order logic">first-order predicate calculus</a> and the simply-typed <a href="/wiki/Polymorphism_(computer_science)" title="Polymorphism (computer science)">polymorphic</a> <a href="/wiki/Lambda_calculus" title="Lambda calculus">lambda calculus</a>, had ML as its metalanguage).</p>/n<p>Today there are several languages in the ML family; the three most prominent are <a href="/wiki/Standard_ML" title="Standard ML">Standard ML</a> (SML), <a href="/wiki/OCaml" title="OCaml">OCaml</a> and <a href="/wiki/F_Sharp_(programming_language)" title="F Sharp (programming language)">F#</a>. Ideas from ML have influenced numerous other languages, like Haskell, <a href="/wiki/Cyclone_programming_language" class="mw-redirect" title="Cyclone programming language">Cyclone</a>, <a href="/wiki/Nemerle" title="Nemerle">Nemerle</a>, <a href="/wiki/ATS_(programming_language)" title="ATS (programming language)">ATS</a>,<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (August 2010)">citation needed</span></a></i>]</sup> and <a href="/wiki/Elm_(programming_language)" title="Elm (programming language)">Elm</a>.<sup id="cite_ref-3" class="reference"><a href="#cite_note-3">[3]</a></sup></p>/n<h2><span class="mw-headline" id="Examples">Examples</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=ML_(programming_language)&amp;action=edit&amp;section=2" title="Edit section: Examples">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>The following examples use the syntax of Standard ML. Other ML dialects like <a href="/wiki/OCaml" title="OCaml">OCaml</a> and <a href="/wiki/F_Sharp_(programming_language)" title="F Sharp (programming language)">F#</a> differ in slightly ways.</p>/n<h3><span class="mw-headline" id="Factorial">Factorial</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=ML_(programming_language)&amp;action=edit&amp;section=3" title="Edit section: Factorial">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<p>The <a href="/wiki/Factorial" title="Factorial">factorial</a> function expressed as pure ML:</p>/n<div class="mw-highlight mw-content-ltr" dir="ltr">/n<pre>/n<span class="kr">fun</span> <span class="nf">fac</span> <span class="p">(</span><span class="mi">0</span> <span class="p">:</span> <span class="n">int</span><span class="p">)</span> <span class="p">:</span> <span class="n">int</span> <span class="p">=</span> <span class="mi">1</span>/n  <span class="p">|</span> <span class="nf">fac</span> <span class="p">(</span><span class="n">n</span> <span class="p">:</span> <span class="n">int</span><span class="p">)</span> <span class="p">:</span> <span class="n">int</span> <span class="p">=</span> <span class="n">n</span> <span class="n">*</span> <span class="n">fac</span> <span class="p">(</span><span class="n">n</span> <span class="n">-</span> <span class="mi">1</span><span class="p">)</span>/n</pre></div>/n<p>This describes the factorial as a recursive function, with a single terminating base case. It is similar to the descriptions of factorials found in mathematics textbooks. Much of ML code is similar to mathematics in facility and syntax.</p>/n<p>Part of the definition shown is optional, and describes the <i>types</i> of this function. The notation E&#160;: t can be read as <i>expression E has type t</i>. For instance, the argument n is assigned type <i>integer</i> (int), and fac (n&#160;: int), the result of applying fac to the integer n, also has type integer. The function fac as a whole then has type <i>function from integer to integer</i> (int -&gt; int), that is, fac accepts an integer as an argument and returns an integer result. Thanks to type inference, the type annotations can be omitted and will be derived by the compiler. Rewritten without the type annotations, the example looks like:</p>/n<div class="mw-highlight mw-content-ltr" dir="ltr">/n<pre>/n<span class="kr">fun</span> <span class="nf">fac</span> <span class="mi">0</span> <span class="p">=</span> <span class="mi">1</span>/n  <span class="p">|</span> <span class="nf">fac</span> <span class="n">n</span> <span class="p">=</span> <span class="n">n</span> <span class="n">*</span> <span class="n">fac</span> <span class="p">(</span><span class="n">n</span> <span class="n">-</span> <span class="mi">1</span><span class="p">)</span>/n</pre></div>/n<p>The function also relies on pattern matching, an important part of ML programming. Note that parameters of a function are not necessarily in parentheses but separated by spaces. When the function's argument is 0 (zero) it will return the integer 1 (one). For all other cases the second line is tried. This is the <a href="/wiki/Recursion" title="Recursion">recursion</a>, and executes the function again until the base case is reached.</p>/n<p>This implementation of the factorial function is not guaranteed to terminate, since a negative argument causes an <a href="/wiki/Infinite_descending_chain" title="Infinite descending chain">infinite descending chain</a> of recursive calls. That is, it can consume lists whose elements have any type, and return lists of the same type.</p>/n<h3><span class="mw-headline" id="Modules">Modules</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=ML_(programming_language)&amp;action=edit&amp;section=5" title="Edit section: Modules">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<p>Modules are ML's system for structuring large projects and libraries.
