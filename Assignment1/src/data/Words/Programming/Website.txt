For the Internet domain <i>.website</i>, see <a href="/wiki/List_of_Internet_top-level_domains" title="List of Internet top-level domains">List of Internet top-level domains</a>.</div>/n<div role="note" class="hatnote navigation-not-searchable">Not to be confused with <a href="/wiki/WebCite" title="WebCite">WebCite</a>.</div>/n<div class="thumb tright">/n<div class="thumbinner" style="width:242px;"><a href="/wiki/File:NASA_website_homepage.jpg" class="image"><img alt="" src="//upload.wikimedia.org/wikipedia/commons/thumb/a/a1/NASA_website_homepage.jpg/240px-NASA_website_homepage.jpg" width="240" height="387" class="thumbimage" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/a/a1/NASA_website_homepage.jpg/360px-NASA_website_homepage.jpg 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/a/a1/NASA_website_homepage.jpg/480px-NASA_website_homepage.jpg 2x" data-file-width="1418" data-file-height="2284" /></a>/n<div class="thumbcaption">/n<div class="magnify"><a href="/wiki/File:NASA_website_homepage.jpg" class="internal" title="Enlarge"></a></div>/nA NASA.gov homepage</div>/n</div>/n</div>/n<p>A <b>website</b><sup id="cite_ref-1" class="reference"><a href="#cite_note-1">[1]</a></sup> is a collection of related <a href="/wiki/Web_page" title="Web page">web pages</a>, including <a href="/wiki/Multimedia" title="Multimedia">multimedia</a> content, typically identified with a common <a href="/wiki/Domain_name" title="Domain name">domain name</a>, and published on at least one <a href="/wiki/Web_server" title="Web server">web server</a>. A website may be accessible via a public <a href="/wiki/Internet_Protocol" title="Internet Protocol">Internet Protocol</a> (IP) network, such as the <a href="/wiki/Internet" title="Internet">Internet</a>, or a private <a href="/wiki/Local_area_network" title="Local area network">local area network</a> (LAN), by referencing a <a href="/wiki/URL" title="URL">uniform resource locator</a> (URL) that identifies the site.</p>/n<p>Websites can have many functions and can be used in various fashions; a website can be a <a href="/wiki/Personal_website" class="mw-redirect" title="Personal website">personal website</a>, a commercial website for a company, a <a href="/wiki/E-Government" class="mw-redirect" title="E-Government">government website</a> or a <a href="/wiki/Nonprofit_organization" title="Nonprofit organization">non-profit organization</a> website. Websites are typically dedicated to a particular topic or purpose, ranging from entertainment and <a href="/wiki/Social_networking" class="mw-redirect" title="Social networking">social networking</a> to providing news and education. All publicly accessible websites collectively constitute the <a href="/wiki/World_Wide_Web" title="World Wide Web">World Wide Web</a>, while private websites, such as a company's website for its employees, are typically a part of an <a href="/wiki/Intranet" title="Intranet">intranet</a>.</p>/n<p>Web pages, which are the <a href="/wiki/Building_block" class="mw-redirect" title="Building block">building blocks</a> of websites, are <a href="/wiki/Document" title="Document">documents</a>, typically composed in <a href="/wiki/Plain_text" title="Plain text">plain text</a> interspersed with formatting instructions of Hypertext Markup Language (<a href="/wiki/HTML" title="HTML">HTML</a>, <a href="/wiki/XHTML" title="XHTML">XHTML</a>). They may incorporate elements from other websites with suitable <a href="/wiki/HTML_anchor" class="mw-redirect" title="HTML anchor">markup anchors</a>. Web pages are accessed and transported with the <a href="/wiki/Hypertext_Transfer_Protocol" title="Hypertext Transfer Protocol">Hypertext Transfer Protocol</a> (HTTP), which may optionally employ encryption (<a href="/wiki/HTTP_Secure" class="mw-redirect" title="HTTP Secure">HTTP Secure</a>, HTTPS) to provide security and privacy for the user. The user's application, often a <a href="/wiki/Web_browser" title="Web browser">web browser</a>, renders the page content according to its HTML markup instructions onto a <a href="/wiki/Computer_monitor" title="Computer monitor">display terminal</a>.</p>/n<p><a href="/wiki/Hyperlink" title="Hyperlink">Hyperlinking</a> between web pages conveys to the reader the <a href="/wiki/Site_map" title="Site map">site structure</a> and guides the navigation of the site, which often starts with a <a href="/wiki/Home_page" title="Home page">home page</a> containing a directory of the site <a href="/wiki/Web_content" title="Web content">web content</a>. Some websites require user registration or <a href="/wiki/Subscription" class="mw-redirect" title="Subscription">subscription</a> to access content. Examples of <a href="/wiki/Paywall" title="Paywall">subscription websites</a> include many business sites, news websites, <a href="/wiki/Academic_journal" title="Academic journal">academic journal</a> websites, gaming websites, file-sharing websites, <a href="/wiki/Internet_forum" title="Internet forum">message boards</a>, web-based <a href="/wiki/Email" title="Email">email</a>, <a href="/wiki/Social_networking" class="mw-redirect" title="Social networking">social networking</a> websites, websites providing real-time <a href="/wiki/Stock_market" title="Stock market">stock market</a> data, as well as sites providing various other services. As of 2017, <a href="/wiki/End_user" title="End user">end users</a> can access websites on a range of devices, including <a href="/wiki/Desktop_computer" title="Desktop computer">desktop</a> and <a href="/wiki/Laptop" title="Laptop">laptop computers</a>, <a href="/wiki/Tablet_computer" title="Tablet computer">tablet computers</a>, <a href="/wiki/Smartphone" title="Smartphone">smartphones</a> and <a href="/wiki/Smart_TV" title="Smart TV">smart TVs</a>.</p>/n<p></p>/n<div id="toc" class="toc">/n<div class="toctitle">/n<h2>Contents</h2>/n</div>/n<ul>/n<li class="toclevel-1 tocsection-1"><a Wide Web">History of the World Wide Web</a></div>/n<div class="thumb tright">/n<div class="thumbinner" style="width:242px;"><a href="/wiki/File:NASA_Website_Homepage_-_April_25,_2015.png" class="image"><img alt="" src="//upload.wikimedia.org/wikipedia/commons/thumb/2/20/NASA_Website_Homepage_-_April_25%2C_2015.png/240px-NASA_Website_Homepage_-_April_25%2C_2015.png" width="240" in April 2015</div>/n</div>/n</div>/n<p>The <a href="/wiki/World_Wide_Web" title="World Wide Web">World Wide Web</a> (WWW) was created in 1990 by the British CERN physicist <a href="/wiki/Tim_Berners-Lee" title="Tim Berners-Lee">Tim Berners-Lee</a>.<sup id="cite_ref-First_page_2-0" class="reference"><a href="#cite_note-First_page-2">[2]</a></sup> On 30 April 1993, <a href="/wiki/CERN" title="CERN">CERN</a> announced that the World Wide Web would be free to use for anyone.<sup id="cite_ref-w3c_3-0" class="reference"><a href="#cite_note-w3c-3">[3]</a></sup> Before the introduction of HTML and HTTP, other protocols such as <a href="/wiki/File_Transfer_Protocol" title="File Transfer Protocol">File Transfer Protocol</a> and the <a href="/wiki/Gopher_protocol" class="mw-redirect" title="Gopher protocol">gopher protocol</a> were used to retrieve individual files from a server. These protocols offer a simple directory structure which the user navigates and chooses files to download. Documents were most often presented as plain text files without formatting, or were encoded in <a href="/wiki/Word_processor" title="Word processor">word processor</a> formats.</p>/n<h2><span class="mw-headline" id="Overview">Overview</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Website&amp;action=edit&amp;section=2" title="Edit section: Overview">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>Websites have many functions and can be used in various fashions; a website can be a <a href="/wiki/Personal_website" class="mw-redirect" title="Personal website">personal website</a>, a commercial website, a <a href="/wiki/E-Government" class="mw-redirect" title="E-Government">government website</a> or a <a href="/wiki/Nonprofit_organization" title="Nonprofit organization">non-profit organization</a> website. Websites can be the work of an individual, a business or other organization, and are typically dedicated to a particular topic or purpose. Any website can contain a <a href="/wiki/Hyperlink" title="Hyperlink">hyperlink</a> to any other website, so the distinction between individual sites, as perceived by the user, can be blurred. Websites are written in, or converted to, HTML (<a href="/wiki/Hyper_Text_Markup_Language" class="mw-redirect" title="Hyper Text Markup Language">Hyper Text Markup Language</a>) and are accessed using a <a href="/wiki/Software" title="Software">software</a> interface classified as a <a href="/wiki/User_agent" title="User agent">user agent</a>. Web pages can be viewed or otherwise accessed from a range of <a href="/wiki/Computer" title="Computer">computer</a>-based and Internet-enabled devices of various sizes, including desktop computers, <a href="/wiki/Laptops" class="mw-redirect" title="Laptops">laptops</a>, tablet computers and smartphones. A website is <a href="/wiki/Web_hosting_service" title="Web hosting service">hosted</a> on a computer system known as a <a href="/wiki/Web_server" title="Web server">web server</a>, also called an HTTP (Hyper Text Transfer Protocol) server. These terms can also refer to the software that runs on these systems which retrieves and delivers the web pages in response to requests from the website's users. <a href="/wiki/Apache_HTTP_Server" title="Apache HTTP Server">Apache</a> is the most commonly used web server software (according to <a href="/wiki/Netcraft" title="Netcraft">Netcraft</a> statistics) and <a href="/wiki/Microsoft" title="Microsoft">Microsoft</a>'s <a href="/wiki/Internet_Information_Services" title="Internet Information Services">IIS</a> is also commonly used. Some alternatives, such as <a href="/wiki/Nginx" title="Nginx">Nginx</a>, <a href="/wiki/Lighttpd" title="Lighttpd">Lighttpd</a>, <a href="/wiki/Hiawatha_(web_server)" title="Hiawatha (web server)">Hiawatha</a> or <a href="/wiki/Cherokee_(web_server)" title="Cherokee (web server)">Cherokee</a>, are fully functional and lightweight.</p>/n<h2><span class="mw-headline" id="Static_website">Static website</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Website&amp;action=edit&amp;section=3" title="Edit section: Static website">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<div role="note" class="hatnote navigation-not-searchable">Main article: <a href="/wiki/Static_web_page" title="Static web page">Static web page</a></div>/n<p>A static website is one that has web pages stored on the server in the format that is sent to a client web browser. It is primarily coded in <a href="/wiki/Hypertext_Markup_Language" class="mw-redirect" title="Hypertext Markup Language">Hypertext Markup Language</a> (HTML); <a href="/wiki/Cascading_Style_Sheets" title="Cascading Style Sheets">Cascading Style Sheets</a> (CSS) are used to control appearance beyond basic HTML. Images are commonly used to effect the desired appearance and as part of the main content. Audio or video might also be considered "static" content if it plays automatically or is generally non-interactive. This type of website usually displays the same information to all visitors. Similar to handing out a printed brochure to customers or clients, a static website will generally provide consistent, standard information for an extended period of time. Although the website owner may make updates periodically, it is a manual process to edit the text, photos and other content and may require basic website design skills and software. Simple forms or marketing examples of websites, such as <i>classic website</i>, a <i>five-page website</i> or a <i>brochure website</i> are often static websites, because they present pre-defined, static information to the user. This may include information about a company and its products and services through text, photos, animations, audio/video, and navigation menus.</p>/n<p>Static websites can be edited using four broad categories of software:</p>/n<ul>/n<li><a href="/wiki/Text_editor" title="Text editor">Text editors</a>, such as <a href="/wiki/Notepad_(Windows)" class="mw-redirect" title="Notepad (Windows)">Notepad</a> or <a href="/wiki/TextEdit" title="TextEdit">TextEdit</a>, where content and HTML markup are manipulated directly within the editor program</li>/n<li><a href="/wiki/WYSIWYG" title="WYSIWYG">WYSIWYG</a> offline editors, such as <a href="/wiki/Microsoft_FrontPage" title="Microsoft FrontPage">Microsoft FrontPage</a> and <a href="/wiki/Adobe_Dreamweaver" title="Adobe Dreamweaver">Adobe Dreamweaver</a> (previously Macromedia Dreamweaver), with which the site is edited using a <a href="/wiki/GUI" class="mw-redirect" title="GUI">GUI</a> and the final HTML markup is generated automatically by the editor software</li>/n<li>WYSIWYG online editors which create media rich online presentation like web pages, widgets, intro, blogs, and other documents.</li>/n<li><a href="/wiki/Web_template_system" title="Web template system">Template-based editors</a> such as <a href="/wiki/IWeb" title="IWeb">iWeb</a> allow users to create and upload web pages to a web server without detailed HTML knowledge, as they pick a suitable template from a palette and add pictures and text to it in a <a href="/wiki/Desktop_publishing" title="Desktop publishing">desktop publishing</a> fashion without direct manipulation of HTML code.</li>/n</ul>/n<p>Static websites may still use <a href="/wiki/Server_side_include" class="mw-redirect" title="Server side include">server side includes</a> (SSI) as an editing convenience, such as sharing a common menu bar across many pages. As the site's behaviour <i>to the reader</i> is still static, this is not considered a dynamic site.</p>/n<h2><span class="mw-headline" id="Dynamic_website">Dynamic website</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Website&amp;action=edit&amp;section=4" title="Edit section: Dynamic website">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<div class="thumb tright">/n<div class="thumbinner" style="width:222px;"><a href="/wiki/File:Server-side_websites_programming_languages.PNG" class="image"><img alt="" src="//upload.wikimedia.org/wikipedia/commons/thumb/8/88/Server-side_websites_programming_languages.PNG/220px-Server-side_websites_programming_languages.PNG" width="220" height="131" class="thumbimage" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/8/88/Server-side_websites_programming_languages.PNG/330px-Server-side_websites_programming_languages.PNG 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/8/88/Server-side_websites_programming_languages.PNG/440px-Server-side_websites_programming_languages.PNG 2x" data-file-width="570" data-file-height="340" /></a>/n<div class="thumbcaption">/n<div class="magnify"><a href="/wiki/File:Server-side_websites_programming_languages.PNG" class="internal" title="Enlarge"></a></div>/nServer-side programming languages repartition on 28 April 2016.</div>/n</div>/n</div>/n<div role="note" class="hatnote navigation-not-searchable">Main articles: <a href="/wiki/Dynamic_web_page" title="Dynamic web page">Dynamic web page</a> and <a href="/wiki/Web_application" title="Web application">Web application</a></div>/n<p>A dynamic website is one that changes or customizes itself frequently and automatically. Server-side dynamic pages are generated "on the fly" by computer code that produces the HTML (CSS are responsible for appearance and thus, are static files). There are a wide range of software systems, such as <a href="/wiki/Common_Gateway_Interface" title="Common Gateway Interface">CGI</a>, <a href="/wiki/Java_Servlets" class="mw-redirect" title="Java Servlets">Java Servlets</a> and <a href="/wiki/Java_Server_Pages" class="mw-redirect" title="Java Server Pages">Java Server Pages</a> (JSP), <a href="/wiki/Active_Server_Pages" title="Active Server Pages">Active Server Pages</a> and <a href="/wiki/ColdFusion" class="mw-redirect" title="ColdFusion">ColdFusion</a> (CFML) that are available to generate <a href="/wiki/Programming_languages_used_in_most_popular_websites" title="Programming languages used in most popular websites">dynamic web systems and dynamic sites</a>. Various <a href="/wiki/Web_application_framework" class="mw-redirect" title="Web application framework">web application frameworks</a> and <a href="/wiki/Web_template_system" title="Web template system">web template systems</a> are available for general-use <a href="/wiki/Programming_language" title="Programming language">programming languages</a> like <a href="/wiki/Perl" title="Perl">Perl</a>, <a href="/wiki/PHP" title="PHP">PHP</a>, <a href="/wiki/Python_(programming_language)" title="Python (programming language)">Python</a> and <a href="/wiki/Ruby_(programming_language)" title="Ruby (programming language)">Ruby</a> to make it faster and easier to create complex dynamic websites.</p>/n<p>A site can display the current state of a dialogue between users, monitor a changing situation, or provide information in some way personalized to the requirements of the individual user. For example, when the front page of a news site is requested, the code running on the web server might combine stored HTML fragments with news stories retrieved from a <a href="/wiki/Database" title="Database">database</a> or another website via <a href="/wiki/RSS" title="RSS">RSS</a> to produce a page that includes the latest information. Dynamic sites can be interactive by using <a href="/wiki/HTML_forms" class="mw-redirect" title="HTML forms">HTML forms</a>, storing and reading back <a href="/wiki/Browser_cookies" class="mw-redirect" title="Browser cookies">browser cookies</a>, or by creating a series of pages that reflect the previous history of clicks. Another example of dynamic content is when a retail website with a database of media products allows a user to input a search request, e.g. for the keyword <a href="/wiki/Beatles" class="mw-redirect" title="Beatles">Beatles</a>. In response, the content of the web page will spontaneously change the way it looked before, and will then display a list of Beatles products like CDs, DVDs and books. <a href="/wiki/Dynamic_HTML" title="Dynamic HTML">Dynamic HTML</a> uses <a href="/wiki/JavaScript" title="JavaScript">JavaScript</a> code to instruct the web browser how to interactively modify the page contents. One way to simulate a certain type of dynamic website while avoiding the performance loss of initiating the dynamic engine on a per-user or per-connection basis, is to periodically automatically regenerate a large series of static pages.</p>/n<h2><span class="mw-headline" id="Multimedia_and_interactive_content">Multimedia and interactive content</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Website&amp;action=edit&amp;section=5" title="Edit section: Multimedia and interactive content">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>Early websites had only text, and soon after, images. Web browser <a href="/wiki/Plug-in_(computing)" title="Plug-in (computing)">plug ins</a> were then used to add audio, video, and interactivity (such as for a <a href="/wiki/Rich_Internet_application" title="Rich Internet application">rich Internet application</a> that mirrors the complexity of a desktop application like a <a href="/wiki/Word_processor" title="Word processor">word processor</a>). Examples of such plug-ins are <a href="/wiki/Microsoft_Silverlight" title="Microsoft Silverlight">Microsoft Silverlight</a>, <a href="/wiki/Adobe_Flash" title="Adobe Flash">Adobe Flash</a>, <a href="/wiki/Adobe_Shockwave" title="Adobe Shockwave">Adobe Shockwave</a>, and <a href="/wiki/Applet" title="Applet">applets</a> written in <a href="/wiki/Java_(programming_language)" title="Java (programming language)">Java</a>. <a href="/wiki/HTML_5" class="mw-redirect" title="HTML 5">HTML 5</a> includes provisions for audio and video without plugins. <a href="/wiki/JavaScript" title="JavaScript">JavaScript</a> is also built into most modern web browsers, and allows for website creators to send code to the web browser that instructs it how to interactively modify page content and communicate with the web server if needed. The browser's internal representation of the content is known as the <a href="/wiki/Document_Object_Model" title="Document Object Model">Document Object Model</a> (DOM) and the technique is known as <a href="/wiki/Dynamic_HTML" title="Dynamic HTML">Dynamic HTML</a>. A 2010-era trend in websites called "responsive design" has given the best of viewing experience as it provides with a device based layout for users. These websites change their layout according to the device or mobile platform thus giving a rich user experience.<sup id="cite_ref-4" class="reference"><a href="#cite_note-4">[4]</a></sup></p>/n<h2><span class="mw-headline" id="Spelling">Spelling</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Website&amp;action=edit&amp;section=6" title="Edit section: Spelling">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>While "web site" was the original spelling (sometimes capitalized "Web site", since "Web" is a proper noun when referring to the <a href="/wiki/World_Wide_Web" title="World Wide Web">World Wide Web</a>), this variant has become rarely used, and "website" has become the standard spelling.
