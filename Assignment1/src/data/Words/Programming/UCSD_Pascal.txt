UCSD Pascal was first released in 1978. It was developed at the <a href="/wiki/University_of_California,_San_Diego" title="University of California, San Diego">University of California, San Diego</a> (UCSD).</p>/n<p></p>/n<div id="toc" p-System</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=UCSD_Pascal&amp;action=edit&amp;section=1" title="Edit section: UCSD Pascal and the p-System">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>In 1978, the University of California, San Diego (UCSD) Institute for Information Systems developed UCSD Pascal to provide students with a common environment that could run on any of the then available <a href="/wiki/Microcomputer" title="Microcomputer">microcomputers</a> as well as campus <a href="/wiki/Digital_Equipment_Corporation" title="Digital Equipment Corporation">DEC</a> <a href="/wiki/PDP-11" title="PDP-11">PDP-11</a> <a href="/wiki/Minicomputer" title="Minicomputer">minicomputers</a>. The operating system became known as UCSD p-System.</p>/n<p>UCSD p-System was one of three operating systems, along with <a href="/wiki/PC_DOS" class="mw-redirect" title="PC DOS">PC DOS</a> and <a href="/wiki/CP/M-86" title="CP/M-86">CP/M-86</a>, that <a href="/wiki/IBM" title="IBM">IBM</a> offered for its original <a href="/wiki/IBM_PC" class="mw-redirect" title="IBM PC">IBM PC</a>.<sup id="cite_ref-williams198201_1-0" class="reference"><a href="#cite_note-williams198201-1">[1]</a></sup> Vendor <a href="/wiki/SofTech,_Inc." title="SofTech, Inc.">SofTech Microsystems</a><sup id="cite_ref-2" class="reference"><a href="#cite_note-2">[2]</a></sup> emphasized p-System's application portability, with virtual machines for 20 CPUs as of the IBM PC's release. It predicted that users would be able to use applications they purchased on future computers running p-System;<sup id="cite_ref-edlinbunnell19820203_3-0" class="reference"><a href="#cite_note-edlinbunnell19820203-3">[3]</a></sup> advertisements called it "the Universal Operating System".<sup id="cite_ref-byte198208_4-0" class="reference"><a href="#cite_note-byte198208-4">[4]</a></sup></p>/n<p><i><a href="/wiki/PC_Magazine" title="PC Magazine">PC Magazine</a></i> denounced UCSD p-System on the IBM PC, stating in a review of <a href="/wiki/Context_MBA" title="Context MBA">Context MBA</a>, written in the language, that it "simply does not produce good code".<sup id="cite_ref-zachmann198306_5-0" class="reference"><a href="#cite_note-zachmann198306-5">[5]</a></sup> The p-System did not sell very well for the IBM PC, because of a lack of applications and because it was more expensive than the other choices. Previously, IBM had offered the UCSD p-System as an option for <a href="/wiki/Displaywriter" class="mw-redirect" title="Displaywriter">Displaywriter</a>, an <a href="/wiki/8086" class="mw-redirect" title="8086">8086</a>-based dedicated word processing machine (not to be confused with IBM's <a href="/wiki/DisplayWrite" class="mw-redirect" title="DisplayWrite">DisplayWrite</a> word processing software). (The Displaywriter's native operating system had been developed completely internally and was not opened for end-user programming.)</p>/n<p>Notable extensions to standard Pascal include separately compilable <i><a href="/wiki/Unit_(Software_Development)" class="mw-redirect" title="Unit (Software Development)">Units</a></i> and a <i>String</i> type. Both of these extensions influenced the design of the <a href="/wiki/Ada_(programming_language)" title="Ada (programming language)">Ada</a> language.<sup id="cite_ref-bowles2004_6-0" class="reference"><a href="#cite_note-bowles2004-6">[6]</a></sup> Some intrinsics were provided to accelerate string processing (e.g. scanning in an array for a particular search pattern); other language extensions were provided to allow the UCSD p-System to be self-compiling and <a href="/wiki/Self-hosting" title="Self-hosting">self-hosted</a>.</p>/n<p>UCSD Pascal was based on a <a href="/wiki/P-code_machine" title="P-code machine">p-code machine</a> architecture. Its contribution to these early <a href="/wiki/Virtual_machines" class="mw-redirect" title="Virtual machines">virtual machines</a> was to extend p-code away from its roots as a compiler <a href="/wiki/Intermediate_language" class="mw-redirect" title="Intermediate language">intermediate language</a> into a full execution environment.<sup class="noprint Inline-Template" style="margin-left:0.1em; white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Please_clarify" title="Wikipedia:Please clarify"><span title="The text near this tag may need clarification or removal of jargon. (October 2010)">clarification needed</span></a></i>]</sup> The UCSD Pascal p-Machine was optimized for the new small microcomputers with addressing restricted to 16-bit (only 64 KB of memory). <a href="/wiki/James_Gosling" title="James Gosling">James Gosling</a> cites UCSD Pascal as a key influence (along with the <a href="/wiki/Smalltalk" title="Smalltalk">Smalltalk</a> virtual machine) on the design of the <a href="/wiki/Java_(programming_language)" title="Java (programming language)">Java</a> virtual machine.<sup id="cite_ref-7" class="reference"><a href="#cite_note-7">[7]</a></sup></p>/n<p>UCSD p-System achieved machine independence by defining a <a href="/wiki/Virtual_machine" title="Virtual machine">virtual machine</a>, called the <i>p-Machine</i> (or pseudo-machine, which many users began to call the "Pascal-machine" like the OS—although UCSD documentation always used "pseudo-machine") with its own <a href="/wiki/Instruction_set" class="mw-redirect" title="Instruction set">instruction set</a> called <a href="/wiki/P-code_machine" title="P-code machine">p-code</a> (or pseudo-code). Urs Ammann, a student of <a href="/wiki/Niklaus_Wirth" title="Niklaus Wirth">Niklaus Wirth</a>, originally presented a p-code in his <a href="/wiki/PhD" class="mw-redirect" title="PhD">PhD</a> thesis,<sup id="cite_ref-8" class="reference"><a href="#cite_note-8">[8]</a></sup> from which the UCSD implementation was derived, the Zurich Pascal-P implementation. The UCSD implementation changed the Zurich implementation to be "byte oriented". The UCSD p-code was optimized for execution of the <a href="/wiki/Pascal_programming_language" class="mw-redirect" title="Pascal programming language">Pascal programming language</a>. Each hardware platform then only needed a p-code interpreter program written for it to port the entire p-System and all the tools to run on it. Later versions also included additional languages that compiled to the p-code base. For example, Apple Computer offered a Fortran Compiler<sup id="cite_ref-9" class="reference"><a href="#cite_note-9">[9]</a></sup> (written by Silicon Valley Software, Sunnyvale California) producing p-code that ran on the Apple version of the p-system. Later, TeleSoft (also located in <a href="/wiki/San_Diego" title="San Diego">San Diego</a>) offered an early <a href="/wiki/Ada_(programming_language)" title="Ada (programming language)">Ada</a> development environment that used p-code and was therefore able to run on a number of hardware platforms including the <a href="/wiki/Motorola_68000" title="Motorola 68000">Motorola 68000</a>, the <a href="/wiki/System/370" class="mw-redirect" title="System/370">System/370</a>, and the <a href="/wiki/Pascal_MicroEngine" title="Pascal MicroEngine">Pascal MicroEngine</a>.</p>/n<p>UCSD p-System shares some concepts with the more current <a href="/wiki/Java_platform" class="mw-redirect" title="Java platform">Java platform</a>. Both use a virtual machine to hide operating system and hardware differences, and both use programs written to that virtual machine to provide <a href="/wiki/Cross-platform" title="Cross-platform">cross-platform</a> support. Likewise both systems allow the virtual machine to be used either as the complete <a href="/wiki/Operating_system" title="Operating system">operating system</a> of the target computer or to run in a "box" under another operating system.</p>/n<p>The UCSD Pascal compiler was distributed as part of a portable <a href="/wiki/Operating_system" title="Operating system">operating system</a>, the p-System.</p>/n<h2><span class="mw-headline" id="History">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=UCSD_Pascal&amp;action=edit&amp;section=2" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>UCSD p-System began around 1974 as the idea of UCSD's <a href="/wiki/Kenneth_Bowles" title="Kenneth Bowles">Kenneth Bowles</a>,<sup id="cite_ref-bowles2004_6-1" class="reference"><a href="#cite_note-bowles2004-6">[6]</a></sup> who believed that the number of new computing platforms coming out at the time would make it difficult for new programming languages to gain acceptance. He based UCSD Pascal on the Pascal-P2 release of the portable <a href="/wiki/Compiler" title="Compiler">compiler</a> from Zurich. He was particularly interested in <a href="/wiki/Pascal_(programming_language)" title="Pascal (programming language)">Pascal</a> as a language to teach programming. UCSD introduced two features that were important improvements on the original Pascal: variable length strings, and "units" of independently compiled code (an idea included into the then-evolving Ada programming language). <a href="/wiki/Niklaus_Wirth" title="Niklaus Wirth">Niklaus Wirth</a> credits the p-System, and UCSD Pascal in particular, with popularizing <a href="/wiki/Pascal_programming_language" class="mw-redirect" title="Pascal programming language">Pascal</a>. It was not until the release of <a href="/wiki/Turbo_Pascal" title="Turbo Pascal">Turbo Pascal</a> that UCSD's version started to slip from first place among Pascal users.</p>/n<p>The Pascal dialect of UCSD Pascal came from the subset of Pascal implemented in Pascal-P2, which was not designed to be a full implementation of the language, but rather "the minimum subset that would self-compile", to fit its function as a bootstrap kit for Pascal compilers. UCSD added strings from BASIC, and several other implementation dependent features. Although UCSD Pascal later obtained many of the other features of the full Pascal language, the Pascal-P2 subset persisted in other dialects, notably Borland Pascal, which copied much of the UCSD dialect.</p>/n<h2><span class="mw-headline" id="Versions">Versions</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=UCSD_Pascal&amp;action=edit&amp;section=3" title="Edit section: Versions">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>There were four versions of UCSD p-code engine, each with several revisions of the p-System and UCSD Pascal. A revision of the p-code engine (i.e., the p-Machine) meant a change to the p-code language, and therefore compiled code is not portable between different p-Machine versions. Each revision was represented with a leading Roman Numeral, while operating system revisions were enumerated as the "dot" number following the p-code Roman Numeral. For example, II.3 represented the third revision of the p-System running on the second revision of the p-Machine.</p>/n<ul>/n<li>Version I</li>/n</ul>/n<dl>/n<dd>/n<dl>/n<dd>Original version, never officially distributed outside of the <a href="/wiki/University_of_California,_San_Diego" title="University of California, San Diego">University of California, San Diego</a>. However, the Pascal sources for both Versions I.3 and I.5 were freely exchanged between interested users. Specifically, the patch revision I.5a was known to be one of the most stable.</dd>/n</dl>/n</dd>/n</dl>/n<ul>/n<li>Version II</li>/n</ul>/n<dl>/n<dd>/n<dl>/n<dd>Widely distributed, available on many early <a href="/wiki/Microcomputer" title="Microcomputer">microcomputers</a>. Numerous versions included <a href="/wiki/Apple_II" title="Apple II">Apple II</a>, <a href="/wiki/Digital_Equipment_Corporation" title="Digital Equipment Corporation">DEC</a> <a href="/wiki/PDP-11" title="PDP-11">PDP-11</a>, <a href="/wiki/Zilog_Z80" title="Zilog Z80">Zilog Z80</a> and <a href="/wiki/MOS_6502" class="mw-redirect" title="MOS 6502">MOS 6502</a> based machines, <a href="/wiki/Motorola_68000" title="Motorola 68000">Motorola 68000</a> and the <a href="/wiki/IBM_PC" class="mw-redirect" title="IBM PC">IBM PC</a> (Version II on the PC was restricted to one 64K <a href="/wiki/Code_segment" title="Code segment">code segment</a> and one 64K stack/heap <a href="/wiki/Data_segment" title="Data segment">data segment</a>; Version IV removed the code segment limit but cost a lot more<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Manual_of_Style/Dates_and_numbers" title="Wikipedia:Manual of Style/Dates and numbers"><span title="This term requires quantification. (October 2010)">quantify</span></a></i>]</sup>).</dd>/n<dd>Project members from this era include/n<ul>/n<li><a href="/wiki/Kenneth_Bowles" title="Kenneth Bowles">Dr Kenneth L Bowles</a></li>/n<li><a href="/wiki/Mark_Allen_(software_developer)" title="Mark Allen (software developer)">Mark Allen</a></li>/n<li>Richard Gleaves</li>/n<li>Richard Kaufmann</li>/n<li>Pete Lawrence</li>/n<li><a href="/wiki/Joel_McCormack" title="Joel McCormack">Joel McCormack</a></li>/n<li>Mark Overgaard</li>/n<li>Keith Shillington</li>/n<li>Roger Sumner</li>/n<li>John Van Zandt</li>/n</ul>/n</dd>/n</dl>/n</dd>/n</dl>/n<ul>/n<li>Version III</li>/n</ul>/n<dl>/n<dd>/n<dl>/n<dd>Custom version written for <a href="/wiki/Western_Digital" title="Western Digital">Western Digital</a> to run on their <a href="/wiki/Pascal_MicroEngine" title="Pascal MicroEngine">Pascal MicroEngine</a> microcomputer. Included support for parallel processes for the first time.</dd>/n</dl>/n</dd>/n</dl>/n<ul>/n<li>Version IV</li>/n</ul>/n<dl>/n<dd>/n<dl>/n<dd>Commercial version, developed and sold by SofTech. Based on Version II; did not include changes from Version III. Did not sell well due to combination of their pricing structure, performance problems due to p-code interpreter, and competition with native operating systems (on top of which it often ran). After SofTech dropped the product, it was picked up by Pecan Systems, a relatively small company formed of p-System users and fans. Sales revived somewhat, due mostly to Pecan's reasonable pricing structure, but the p-System and UCSD Pascal gradually lost the market to native operating systems and compilers. Available for the <a href="/wiki/TI-99/4" class="mw-redirect" title="TI-99/4">TI-99/4</a> equipped with p-code card and <a href="/wiki/Commodore_PET" title="Commodore PET">Commodore PET</a>.</dd>/n</dl>/n</dd>/n</dl>/n<h2><span class="mw-headline" id="Further_use">Further use</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=UCSD_Pascal&amp;action=edit&amp;section=4" title="Edit section: Further use">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>The <a href="/wiki/Corvus_Systems" title="Corvus Systems">Corvus Systems</a> computer used USCD pascal for all its user software. The "innovative concept" of the Constellation OS was to run Pascal (interpretively or compiled) and include all common software in the manual, so users could modify as needed while using (open source was the default in that era).</p>/n<h2><span class="mw-headline" id="See_also">See also</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=UCSD_Pascal&amp;action=edit&amp;section=5" title="Edit section: See also">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<ul>/n<li><a href="/wiki/P-code_machine" title="P-code machine">P-code machine</a></li>/n</ul>/n<h2><span class="mw-headline" id="Notes">Notes</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=UCSD_Pascal&amp;action=edit&amp;section=6" title="Edit section: Notes">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<div class="reflist" style="list-style-type: decimal;">/n<div class="mw-references-wrap">/n<ol class="references">/n<li id="cite_note-williams198201-1"><span class="mw-cite-backlink"><b><a href="#cite_ref-williams198201_1-0">^</a></b></span> <span class="reference-text"><cite class="citation news">Williams, Gregg (January 1982). <a rel="nofollow" class="external text" href="https://archive.org/stream/byte-magazine-1982-01/1982_01_BYTE_07-01_The_IBM_Personal_Computer#page/n37/mode/2up">"A Closer Look at the IBM Personal Computer"</a>. <i>BYTE</i>. p.&#160;36<span class="reference-accessdate">.
