(programming language)">Scratch</a> programming language</div>/n</div>/n</div>/n<p>In <a href="/wiki/Computing" title="Computing">computing</a>, a <b>visual programming language</b> (<b>VPL</b>) is any <a href="/wiki/Programming_language" title="Programming language">programming language</a> that lets users create <a href="/wiki/Computer_program" title="Computer program">programs</a> by manipulating program elements graphically rather than by specifying them textually.<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">[1]</a></sup><sup id="cite_ref-2" class="reference"><a href="#cite_note-2">[2]</a></sup> A VPL allows programming with visual expressions, spatial arrangements of text and graphic symbols, used either as elements of <a href="/wiki/Syntax" title="Syntax">syntax</a> or <a href="/wiki/Secondary_notation" title="Secondary notation">secondary notation</a>. For example, many VPLs (known as <i>dataflow</i> or <i>diagrammatic programming</i>)<sup id="cite_ref-3" class="reference"><a href="#cite_note-3">[3]</a></sup> are based on the idea of "boxes and arrows", where boxes or other screen objects are treated as entities, connected by arrows, lines or arcs which represent relations.</p>/n<p></p>/n<div id="toc" class="toc">/n<div class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Visual_programming_language&amp;action=edit&amp;section=1" title="Edit section: Definition">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>VPLs may be further classified, according to the type and extent of visual expression used, into icon-based languages, form-based languages, and diagram languages. Visual programming environments provide graphical or iconic elements which can be manipulated by users in an interactive way according to some specific spatial grammar for program construction.</p>/n<p>A visually transformed language is a non-visual language with a superimposed visual representation. Naturally visual languages have an inherent visual expression for which there is no obvious textual equivalent.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (April 2010)">citation needed</span></a></i>]</sup></p>/n<p>Current developments try to integrate the visual programming approach with <a href="/wiki/Dataflow_programming" title="Dataflow programming">dataflow programming</a> languages to either have immediate access to the <a href="/wiki/Program_state" class="mw-redirect" title="Program state">program state</a>, resulting in online debugging, or automatic program generation and documentation. Dataflow languages also allow <a href="/wiki/Automatic_parallelization" title="Automatic parallelization">automatic parallelization</a>, which is likely to become one of the greatest programming challenges of the future.<sup id="cite_ref-4" class="reference"><a href="#cite_note-4">[4]</a></sup></p>/n<p>The "visual languages" (<a href="/wiki/Visual_Basic_.NET" title="Visual Basic .NET">Visual Basic</a>, <a href="/wiki/C_Sharp_(programming_language)" title="C Sharp (programming language)">Visual C#</a>, <a href="/wiki/J_Sharp" title="J Sharp">Visual J#</a>, etc.) of the <a href="/wiki/Microsoft_Visual_Studio" title="Microsoft Visual Studio">Microsoft Visual Studio</a> IDE are not visual programming languages. All of these languages are textual and not graphical.
